## BAGIAN 1: KONTEKS — APA ITU VIBE CODING DAN MENGAPA INI PENTING

Vibe coding adalah pendekatan membangun aplikasi dengan mendeskripsikan apa yang kamu inginkan dalam bahasa sehari-hari, lalu AI yang mengerjakan teknisnya. Tidak perlu hafal sintaks pemrograman. Tidak perlu tahu cara setup server. Kamu cukup punya ide, visi, dan kemampuan berkomunikasi dengan AI secara efektif.

Ini bukan lagi sekadar tren. Survei 2025 menunjukkan 44% non-technical founders kini membangun prototipe mereka sendiri menggunakan AI coding tools, bukan menyewa developer. Pasar vibe coding sendiri bernilai $4.7 miliar di 2025 dan diproyeksikan mencapai $12.3 miliar pada 2027 — tumbuh 38% per tahun.

Artinya, ada jutaan orang yang punya ide produk digital, punya passion, punya kebutuhan bisnis — tapi selama ini terhalang oleh satu hal: tidak bisa coding. Vibe coding menghapus halangan itu.

Target market sebuah vibe coding course yang ideal adalah persis orang-orang ini: bukan IT background, tapi punya banyak ide dan motivasi. Bisa jadi seorang guru yang ingin membuat platform belajar, pengusaha kuliner yang ingin app pemesanan, HRD yang ingin sistem tracking karyawan, atau freelancer yang ingin menambah layanan ke kliennya.

Pertanyaannya bukan lagi "apakah mereka bisa belajar vibe coding?" — jawabannya sudah jelas bisa. Pertanyaannya adalah: **tools mana yang paling tepat untuk mengajarkan mereka?**

---

## BAGIAN 2: MEMAHAMI LANSKAP — DUA KATEGORI BESAR

Sebelum membandingkan tools satu per satu, penting untuk memahami bahwa kelima tools yang akan dibahas tidak bermain di arena yang sama. Mereka terbagi dalam dua kategori yang berbeda secara fundamental:

**Kategori 1 — No-Code / Full-Stack App Builders**
Kamu deskripsikan ide → AI bangun appnya di browser → app langsung bisa dipakai. Tidak ada kode yang perlu dilihat, tidak ada setup yang perlu dilakukan. Contoh: Lovable, Emergent.sh.

**Kategori 2 — AI-Native Code Editors / Agentic IDEs**
Kamu bekerja di dalam code editor. AI membantu menulis, merefaktor, mendebug, dan mengorkestrasikan kode. Tetap ada exposure ke codebase, tapi AI mengerjakan sebagian besar kerja beratnya. Contoh: Cursor, Claude Code, Antigravity IDE.

Memahami perbedaan ini adalah kunci untuk mengevaluasi tools dengan adil. Membandingkan Lovable dengan Claude Code itu seperti membandingkan Canva dengan Adobe Photoshop — keduanya untuk "desain", tapi untuk pengguna yang sangat berbeda dan dengan filosofi yang sangat berbeda.

### 3.1 — EMERGENT.SH

**Gambaran Umum**

Emergent.sh adalah platform vibe coding full-stack berbasis web yang dibangun oleh kakak-beradik Mukund Jha (CEO) dan Madhav Jha (CTO) — mantan engineer dari Google, Amazon, dan Dropbox — dan didukung oleh Y Combinator. Didirikan 2024, per 2026 sudah memiliki lebih dari 6 juta builder dari 190 negara.

Tagline mereka menangkap esensinya dengan tepat: kamu adalah CEO, Emergent adalah tim engineering-mu. Kamu bicara dalam bahasa manusia, Emergent yang mengeksekusi.

**Cara Kerja**

Emergent menggunakan sistem multi-agent di mana berbagai AI agent bekerja layaknya tim developer sungguhan. Ada agent yang merencanakan arsitektur, ada yang mengerjakan frontend, ada yang mengerjakan backend, ada yang melakukan testing. Semua terjadi secara otonom di background.

**Fitur Utama**
- Build web app dan mobile app (frontend + backend) dari natural language prompt
- Authentication, database, dan hosting sudah termasuk out of the box
- Testing agent otomatis yang memvalidasi UI, API calls, database interactions, dan integrasi sebelum deploy
- GitHub integration — kode bisa diexport dan dilanjutkan di tempat lain
- Visual editor untuk penyesuaian styling
- Deploy langsung, tidak perlu setup server sendiri

**Pricing (2026)**
- Free: 10 credits/bulan
- Standard: $20/bulan — 100 credits
- Pro: $200/bulan — 750 credits
- Team: $300/bulan
- Top-up credits: $10 untuk 50 credits (tidak expired, lebih fleksibel)
- Catatan penting: tidak ada tier menengah antara $20 dan $200 — ini gap yang besar

**PROS**
- Sangat ramah pemula — proses terasa effortless: describe → refine → publish
- Mendukung web dan mobile app dalam satu platform
- Multi-agent architecture menghasilkan output yang lebih kompleks dibanding single-agent
- Deploy otomatis, tidak perlu pengetahuan DevOps sama sekali
- App sederhana bisa selesai dalam waktu kurang dari 10 menit
- Kode bisa diexport via GitHub — tidak terkunci permanen di ekosistem Emergent
- Rated 4.7/5 untuk overall value oleh reviewer independen

**CONS**
- Harga melompat drastis dari $20 ke $200 — tidak ada middle tier
- Credits tidak rollover — tidak terpakai bulan ini, hangus
- Pengguna non-technical yang belum terampil prompting akan cepat kehabisan credits
- Beberapa user melaporkan kehilangan kode saat ada masalah teknis (per Product Hunt reviews 2026)
- Debugging sangat bergantung pada AI platform — kamu tidak punya akses langsung ke codebase untuk fix manual
- Kompleks logic dan custom backend masih terbatas

**Cocok Untuk**
Non-technical founders yang ingin hasil cepat, MVP validation, internal tools sederhana, orang yang sama sekali tidak ingin menyentuh kode, dan project yang butuh web + mobile sekaligus.

### 3.2 — CURSOR

**Gambaran Umum**

Cursor adalah code editor berbasis VS Code yang dibangun ulang dari nol untuk era AI. Dibuat oleh tim mantan peneliti OpenAI di perusahaan bernama Anysphere. Per Maret 2026, Cursor bernilai $29.3 miliar dengan Annual Recurring Revenue $2 miliar, menjadikannya tool developer AI paling bernilai dalam sejarah.

Cursor bukan no-code tool. Ini adalah senjata utama developer profesional yang ingin produktivitas mereka melonjak drastis.

**Cara Kerja**

Cursor mengindeks seluruh proyek kamu. Ketika kamu membuat permintaan, ia memahami seluruh fungsi, tipe data, pola, dan dependency di semua file — bukan hanya file yang sedang terbuka. Ini yang membedakannya dari autocomplete biasa: AI benar-benar "mengerti" projectmu secara holistik.

**Fitur Utama**
- Tab autocomplete dengan Supermaven — tercepat dan terakurat di industri per 2026
- Agent mode: deskripsikan sebuah fitur, AI update semua file yang relevan sekaligus
- Background agents: jalankan task paralel di cloud sementara kamu mengerjakan hal lain
- Codebase-aware chat: tanya pertanyaan tentang seluruh project, bukan hanya satu file
- Multi-model support: Claude Opus 4, GPT-4o, o3, Gemini 2.5 Pro — pilih sesuai kebutuhan
- Full VS Code compatibility: semua extension VS Code tetap bisa dipakai

**Pricing (2026)**
- Hobby (Free): terbatas, hanya untuk eksperimen
- Pro: $20/bulan — usage-based credits
- Business: $40/bulan/seat — tambah compliance dan admin controls
- Catatan: Ada kontroversi pricing Juni 2025 di mana perubahan ke credit-based memotong jumlah request efektif dari ~500 ke ~225/bulan. CEO Cursor meminta maaf secara publik.

**PROS**
- Terbaik untuk developer yang menulis kode setiap hari — ROI terasa dalam satu minggu pertama
- Codebase awareness paling dalam di kelasnya
- Supermaven autocomplete: kecepatan dan akurasi tertinggi di industri
- Agent mode mengkompresi pekerjaan dari jam menjadi menit untuk task rutin
- Background agents memungkinkan workflow paralel yang tidak dimiliki tool lain
- Fondasi VS Code: biaya switching sangat rendah untuk pengguna VS Code

**CONS**
- Bukan no-code — tetap membutuhkan pemahaman kode dasar minimal
- Tidak menangani deployment, database, atau infrastructure secara bawaan
- Credit system membingungkan; pengguna berat perlu aktif memonitor penggunaan
- Performa menurun pada proyek sangat besar (monorepo jutaan baris)
- Tidak support JetBrains (IntelliJ, PyCharm) atau Vim/Emacs

**Cocok Untuk**
Developer profesional, tim engineering yang butuh AI-native workflow. Tidak cocok untuk orang yang sama sekali tidak mau menyentuh kode.

### 3.3 — CLAUDE CODE

**Gambaran Umum**

Claude Code adalah AI coding assistant berbasis terminal (command line) buatan Anthropic — perusahaan yang juga membuat model AI Claude. Bukan chatbot yang menyarankan potongan kode, melainkan autonomous agent yang membaca seluruh codebase, menulis kode lintas banyak file, menjalankan perintah terminal, dan melakukan commit ke Git.

Per pertengahan 2026, Claude Code menjadi tool coding AI paling banyak dipakai oleh engineer profesional, dengan skor 80.8% di benchmark SWE-bench Verified — tertinggi kedua di industri.

**Cara Kerja**

Claude Code berjalan di terminalmu, bukan di browser atau GUI. Ia terhubung langsung ke file system, Git, dan pipeline yang sudah ada. Ketika kamu memberi instruksi, ia membaca seluruh repo, membuat rencana, mengeksekusi, dan meminta approval sebelum melakukan operasi yang bersifat destruktif.

**Fitur Utama**
- Terminal-first: berjalan di CLI, terintegrasi sempurna dengan workflow Git yang sudah ada
- Plan mode dengan extended thinking untuk task kompleks
- Checkpoints untuk keamanan state kode — bisa rollback kapan saja
- Codebase-wide understanding tanpa perlu pilih file secara manual
- Issue-to-PR workflow: baca GitHub issue, tulis kode, buka PR siap-merge
- Routines: konfigurasi task sekali, jalankan terjadwal otomatis
- Context window sangat besar: 200K token (Pro) hingga 500K token (Enterprise)

**Pricing (2026)**
- Free: tidak termasuk Claude Code sama sekali
- Pro: $20/bulan — termasuk Claude Code via terminal
- Max: $100–$200/bulan — usage jauh lebih tinggi, akses model terkuat (Opus 4.6)
- Team Premium: $100/seat, minimum 5 seat
- API: pay-per-token, rata-rata developer menghabiskan sekitar $6/hari

**PROS**
- Kekuatan reasoning terdalam untuk architectural planning dan refactoring kompleks
- Bekerja dengan workflow Git yang sudah ada tanpa perlu pindah IDE
- Model Claude Opus 4.6 unggul dalam penalaran panjang, debugging, dan arsitektur
- Context window terbesar di kelasnya untuk membaca repo besar sekaligus
- Banyak developer menyebutnya sebagai "junior engineer pertama yang benar-benar terasa seperti bekerja dengan manusia"

**CONS**
- Terminal-based: kurva belajar sangat tinggi, sama sekali tidak intuitif bagi non-developer
- Tidak ada visual UI untuk melihat perubahan (kecuali pakai VS Code extension)
- Harga dari $20 ke $100 adalah lompatan yang sangat besar
- Untuk pemula absolut: tidak cocok sama sekali

**Cocok Untuk**
Developer senior dan software engineer profesional yang ingin partner AI paling canggih untuk sistem production-grade. Bukan untuk non-technical users dalam kondisi apapun.

### 3.4 — LOVABLE

**Gambaran Umum**

Lovable adalah AI-powered app builder berbasis web yang mengkonversi natural language prompt menjadi web application lengkap. Termasuk kategori "vibe coding" murni — describe ide, platform generate full application termasuk interface, backend, dan database. Per 2026, Lovable 2.0 telah dirilis dengan peningkatan signifikan di context handling dan backend logic.

**Cara Kerja**

Kamu chat dengan Lovable, describe apa yang kamu mau, dan Lovable menggenerate kode React/TypeScript yang nyata — bukan template, tapi real code yang bisa dimaintain. Kamu bisa iterasi lewat chat, lihat preview langsung, dan deploy dengan satu klik.

**Fitur Utama**
- Generate full-stack web app dari prompt (frontend + database via Supabase)
- Supabase integration terbaik di kelasnya untuk database dan authentication
- Visual editing interface: ubah layout, spacing, warna, typography secara visual
- GitHub sync: download kode, lanjutkan di tempat lain
- One-click deploy — tidak perlu pengetahuan DevOps
- Preview real-time saat membangun
- Kode menggunakan React/TypeScript yang clean dan maintainable

**Pricing (2026)**
- Free: 5 credits/hari (maksimum 150 credits/bulan)
- Pro: $25/bulan — 100 credits
- Tiap generation prompt = 1 credit; app sederhana biasanya butuh 5–15 credits

**PROS**
- Terbaik di kelasnya untuk membangun full-stack web app dengan cepat
- Supabase integration sangat solid — authentication dan database langsung jadi
- Kode yang dihasilkan bersih — developer bisa lanjutkan secara manual jika perlu
- Ideal untuk MVP, SaaS prototype, demo untuk investor
- UI paling intuitif di kategori no-code app builder
- Komunitas aktif, tutorial dan resource sangat banyak tersedia

**CONS**
- Web-only — tidak bisa build native mobile apps (iOS/Android) sama sekali
- Prompt engineering butuh latihan — hasil sangat bergantung pada kualitas prompt
- Complex logic dan multi-step workflows masih sering menghasilkan output yang tidak sesuai
- Credits cepat habis jika tidak disiplin dalam prompting
- Debugging ketika AI salah bisa frustrating — sulit menemukan akar masalah tanpa akses kode

**Cocok Untuk**
Non-technical founders yang fokus pada web app, entrepreneur yang ingin validasi ide cepat, product teams yang ingin prototype berbasis web, siapa pun yang ingin membuat SaaS sederhana atau internal tool web tanpa menyentuh kode.

### 3.5 — ANTIGRAVITY IDE

**Gambaran Umum**

Antigravity adalah IDE generasi baru buatan Google yang diluncurkan bersamaan dengan Gemini 3 pada November 2025. Dibangun dari nol (bukan sekadar plugin), Antigravity adalah platform agentic-first di mana developer bertransisi dari "penulis kode" menjadi "manajer agent." Google mengakuisisi tim Windsurf pada pertengahan 2025, dan Antigravity terasa seperti Windsurf yang jauh lebih ambisius dengan integrasi multi-agent yang jauh lebih dalam.

Pada Google I/O 2026 (19 Mei 2026), Antigravity 2.0 diluncurkan sebagai desktop app, CLI, SDK, dan managed agent service sekaligus — menegaskan posisinya sebagai platform development lengkap, bukan sekadar IDE.

**Cara Kerja**

Antigravity memiliki dua surface utama:

- **Editor Surface**: Interface mirip VS Code, tapi dengan AI yang sangat terintegrasi. Tab completion, natural language commands, semua VS Code extension bisa dipakai.
- **Manager View (Mission Control)**: Tampilan khusus untuk mengorkestrasikan beberapa agent secara paralel di beberapa workspace sekaligus. Satu-satunya IDE dengan fitur ini per 2026.

Agent-agent ini bisa mengedit file, menjalankan command terminal, membuka browser internal, mengklik user flow, mengambil screenshot sebagai bukti fitur bekerja — semua secara otonom dan bisa dipantau secara visual.

**Fitur Utama**
- Multi-agent orchestration via Manager View — eksklusif, tidak dimiliki IDE manapun
- Built-in browser automation: agent bisa membuka Chromium, navigasi ke dev server, klik, dan screenshot
- Self-verifying agents: agent menguji hasil pekerjaannya sendiri sebelum menyerahkan ke kamu
- Artifacts: structured output dari setiap agent, bisa dikomentari langsung layaknya Google Docs
- 1 juta token context window pada paket Pro — bisa membaca seluruh codebase besar sekaligus
- Support beberapa model: Gemini 3 Pro, Gemini 3.5 Flash, Claude Sonnet 4.5, GPT-OSS
- Agent Skills: kapabilitas agent yang bisa dikustomisasi per project
- Secure Mode untuk kontrol keamanan
- Checkpoint system: bisa rollback ke state sebelumnya kapan saja
- Kompatibel dengan semua VS Code extension (via Open VSX)

**Pricing (2026)**
- Free Preview: gratis dengan akses penuh ke Gemini 3 Pro (sedang berjalan)
- Pro: $20/bulan — bagian dari Google AI Pro subscription
- Ultra 5x: $100/bulan — bagian dari Google AI Ultra 5x subscription
- Ultra 20x: $200/bulan — bagian dari Google AI Ultra 20x subscription

**PROS**
- Satu-satunya IDE dengan multi-agent orchestration bawaan yang bisa dipantau secara visual
- Browser automation bawaan — agent verifikasi fitur langsung di browser tanpa tools tambahan
- Self-verifying: agent menguji output-nya sendiri, mengurangi bug yang lolos ke tahap berikutnya
- Context window 1 juta token — mampu memproses codebase besar tanpa kehilangan konteks
- Fondasi VS Code — transisi mudah dari atau ke Cursor/Windsurf
- Manager View membuat proses berpikir AI terlihat — sangat bermanfaat untuk teaching dan learning
- Antigravity 2.0 sudah punya CLI dan SDK — skill yang sangat transferable ke dunia profesional

**CONS**
- Masih relatif baru — ekosistem dan komunitas lebih kecil dari Cursor
- Credit system belum transparan — Google sendiri belum mendokumentasikan dengan jelas berapa included credits per tier
- Ada laporan user yang tiba-tiba mengalami quota lockout beberapa hari
- Risiko "Google Graveyard" — keberlanjutan bergantung pada prioritas strategis Google
- Open VSX only — tidak bisa akses official VS Code Extension Marketplace
- Tidak support JetBrains sama sekali

## BAGIAN 5: MENGAPA ANTIGRAVITY IDE ADALAH PILIHAN PALING TEPAT UNTUK VIBE CODING COURSE

Ini bukan soal Antigravity adalah tools terbaik secara absolut untuk semua orang. Claude Code lebih powerful untuk engineering profesional. Lovable lebih mudah untuk pemula absolut. Cursor lebih mature dari segi ekosistem.

Tapi untuk konteks mengajarkan non-technical people membangun produk digital dengan pendekatan semi vibe coding, Antigravity adalah pilihan yang paling masuk akal secara strategis — dan berikut adalah penjelasan mendalam mengapa.

---

### Alasan 1: Ekosistem Google = Nilai Terbesar per Rupiah

Ini adalah argumen finansial yang sangat kuat dan sering dilewatkan orang.

Ketika kamu berlangganan Google AI Pro ($20/bulan) atau Google AI Ultra 5x ($100/bulan), yang kamu dapatkan bukan hanya Antigravity — tapi satu ekosistem penuh yang dikonfirmasi resmi di Google I/O 2026:

**Google AI Pro (~$20/bulan) termasuk:**
- Antigravity IDE dengan expanded rate limits
- NotebookLM dengan expanded limits (research assistant berbasis dokumen)
- Google Flow (AI video generation)
- Gemini in Docs, Sheets, dan Gmail
- 2 TB Google Drive storage
- Gemini CLI
- Jules (async coding agent terintegrasi GitHub)
- Google Search AI Mode
- Expanded access ke Gemini 3 Pro model

**Google AI Ultra 5x ($100/bulan) termasuk semua di atas, PLUS:**
- 5x usage limits lebih tinggi di semua produk
- 20 TB Google Drive storage
- Family sharing hingga 5 orang
- YouTube Premium individual
- Google Developer Program Premium (termasuk $40/bulan Google Cloud credits)
- Project Mariner: 10 browser tasks paralel untuk otomasi
- Priority access ke Antigravity dengan model terbaru
- Higher limits di NotebookLM, Google AI Studio, dan semua produk Google AI

Bandingkan ini dengan Cursor Pro atau Lovable Pro di $20–$25/bulan yang hanya memberikan satu tools saja. Untuk murid yang baru memulai perjalanan digital mereka, Google AI Pro adalah paket paling value-for-money yang ada di pasaran saat ini — mereka membayar harga yang sama tapi mendapatkan ekosistem kerja lengkap.

Lebih dari itu: murid yang sudah punya Google One atau berencana upgrade bisa mendapatkan semua ini dalam satu tagihan yang sudah familiar. Tidak perlu setup akun baru, tidak perlu kelola banyak langganan terpisah.

---

### Alasan 2: "Semi Vibe Coding" adalah Pendekatan Pedagogis yang Lebih Baik

Lovable dan Emergent memang lebih mudah — tapi kemudahan itu datang dengan biaya tersembunyi: murid tidak memahami apa yang mereka bangun.

Antigravity memposisikan dirinya di sweet spot antara dua ekstrem:

- Terlalu mudah (Lovable/Emergent): kode terjadi di background, murid hanya klik dan describe, tidak memahami struktur apa yang dibangun, debugging menjadi black box
- Terlalu sulit (Cursor/Claude Code): butuh coding knowledge yang solid, kurva belajar terlalu curam untuk pemula non-IT

Dengan Antigravity, murid berada di posisi "Architect" atau "Mission Controller" — mereka melihat codebase, mereka berinteraksi dengan struktur file, tapi AI yang mengerjakan detail teknis. Ini menghasilkan beberapa hal penting:

- Murid belajar cara berpikir terstruktur dan logis dalam mendefinisikan masalah
- Murid menghargai proses iterasi: plan → build → test → refine
- Murid memahami kenapa bug terjadi, bukan hanya tau cara fix-nya via prompt
- Murid memiliki fondasi untuk terus berkembang setelah course selesai

Ini bukan "pure vibe coding" — ini "semi vibe coding" yang jauh lebih berharga jangka panjang. Murid yang lulus dengan pemahaman ini punya skill yang jauh lebih solid dan marketable dibanding murid yang hanya bisa klik tombol di Lovable.

---

### Alasan 3: Kontrol Penuh atas Codebase

Di Lovable dan Emergent, kamu pada dasarnya menyewa lingkungan terkontrol milik mereka. Kode dihasilkan di server mereka, disimpan di ekosistem mereka, dan setiap interaksi — termasuk fixing bug — dilakukan melalui antarmuka mereka.

Konsekuensinya nyata:
- Setiap percobaan fix bug = token/credit terpakai
- Ketika ada issue yang sangat spesifik, kamu sangat bergantung pada kemampuan AI platform itu
- Customization di luar kemampuan platform menjadi sangat sulit atau tidak mungkin
- Jika kamu pindah platform, proses migrasi bisa sangat menyulitkan

Di Antigravity, file ada di local environment murid. Ini berarti:
- Murid bisa inspect kode kapan saja tanpa biaya tambahan
- Rollback via checkpoint system atau Git bisa dilakukan di luar sesi AI
- Project bisa dipindahkan ke tools lain (Cursor, VS Code, Claude Code) tanpa kehilangan apapun
- Debugging lebih presisi karena murid tahu struktur file-nya
- Ada rasa kepemilikan nyata atas apa yang dibangun

---

### Alasan 4: Jebakan Token di Lovable & Emergent — Masalah Nyata untuk Learner

Ini adalah salah satu keluhan paling konsisten di komunitas vibe coding, dan sangat relevan untuk konteks mengajar.

Di Lovable dan Emergent, model bisnisnya adalah per-credit atau per-prompt. Setiap interaksi dengan AI mengkonsumsi token atau credit, termasuk:
- Prompt yang hasilnya salah dan perlu diulang
- Prompt koreksi karena AI salah interpretasi
- Prompt debugging karena output tidak sesuai harapan
- Context window yang terus bertambah besar seiring project berkembang

Masalahnya: orang non-technical yang sedang belajar secara alamiah akan membuat banyak prompt yang tidak optimal. Mereka belum tahu cara mendefinisikan masalah dengan presisi. Mereka akan trial-and-error. Dan setiap trial = credit terpakai.

Untuk app yang cukup kompleks, 100 credits Emergent ($20/bulan) bisa habis sebelum app selesai. Murid yang sudah bayar kursus, sudah semangat, tiba-tiba kuotanya habis di tengah jalan — ini bukan pengalaman belajar yang baik.

Di Antigravity dengan subscription Google, model usage-nya berbeda — flat subscription dengan quota yang jauh lebih predictable untuk kebutuhan pembelajaran. Dan dengan strategi penggunaan model yang tepat (Gemini Flash untuk iterasi dan eksperimen, Gemini Pro untuk final build), murid bisa mengoptimalkan penggunaan secara signifikan karena Gemini Flash sekitar 8x lebih hemat token per penggunaan dibanding Gemini Pro.

---

### Alasan 5: Context Window 1 Juta Token

Antigravity Pro hadir dengan context window 1 juta token — terbesar di antara semua tools dalam perbandingan ini. Artinya AI bisa "membaca" seluruh codebase project yang sudah berkembang tanpa kehilangan konteks.

Ini kritis untuk konteks pembelajaran: project yang dimulai dari kecil akan berkembang. Makin besar project, makin penting AI bisa memahami keseluruhan struktur sebelum membuat perubahan. Tools dengan context window kecil akan mulai "lupa" bagian awal project ketika project sudah cukup besar, menghasilkan perubahan yang konflik dengan kode yang sudah ada.

---

### Alasan 6: Manager View sebagai Alat Pengajaran yang Unik

Ini adalah argumen pedagogis yang sering diabaikan. Manager View di Antigravity membuat proses berpikir AI menjadi visible secara eksplisit.

Ketika murid melihat beberapa agent bekerja paralel — satu agent merencanakan arsitektur, satu menulis frontend, satu mengerjakan backend, satu melakukan testing — mereka secara intuitif memahami bagaimana sistem software dibangun. Mereka melihat bahwa software bukan satu blok besar tapi kumpulan komponen yang berinteraksi.

Ini jauh lebih edukatif dibanding Lovable atau Emergent di mana semua terjadi di background dan murid hanya melihat hasil akhir. Manager View menjadi "jendela" ke dalam proses engineering yang sesungguhnya.

---

### Alasan 7: Portabilitas Skill

Antigravity dibangun di atas VS Code fork. Murid yang belajar di Antigravity otomatis familiar dengan interface yang dipakai oleh lebih dari 60% developer di seluruh dunia.

Ketika mereka nanti ingin masuk lebih dalam ke dunia pengembangan profesional — atau ketika mereka sudah cukup advance dan ingin bermigrasi ke Cursor atau Claude Code — transisinya akan sangat mulus. Muscle memory yang sudah terbentuk di Antigravity langsung applicable.

Sebaliknya, murid yang belajar di Lovable atau Emergent belajar interface yang sangat proprietary. Kalau platform itu berubah atau tutup, skill mereka tidak bisa ditransfer ke tools lain.

---

### Alasan 8: Jules sebagai Pair Programmer Bawaan

Termasuk dalam subscription Google AI adalah Jules — async coding agent yang terintegrasi langsung dengan GitHub. Murid bisa assign task ke Jules di background, dan ketika mereka kembali, sudah ada pull request yang siap di-review.

Ini bukan feature kecil. Ini mengajarkan murid workflow profesional dari hari pertama: bagaimana mendelegasikan task ke AI agent, bagaimana mereview perubahan yang dibuat AI, bagaimana bekerja asinkron dengan "tim" yang bisa bekerja kapan saja termasuk saat mereka tidur.

---

### Alasan 9: Google Play Console Integration untuk Mobile

Di Google I/O 2026, Google mengumumkan Google AI Studio mendapatkan native Android app building dari single prompt, plus direct Google Play Console integration — kamu bisa publish ke test track tanpa meninggalkan ekosistem Google.

Untuk murid yang ingin build mobile app dan publish ke Play Store, ekosistem Google memberikan jalur end-to-end yang tidak dimiliki Lovable sama sekali (yang web-only) dan tidak ada di tools lain secara native.

## BAGIAN 6: SATU CAVEAT YANG PERLU DIANTISIPASI SEBAGAI INSTRUKTUR

Tidak ada tools yang sempurna, dan jujur tentang kelemahan adalah bagian dari teaching yang baik.

**Credit system Antigravity masih belum transparan.** Google sendiri belum mendokumentasikan secara publik berapa credits yang included per tier, dan disclaimer resmi di dokumentasi mereka menyatakan "specified rate limits are not guaranteed." Ada laporan dari komunitas developer tentang quota yang tiba-tiba habis dan menyebabkan lockout akses selama beberapa hari.

Sebagai instruktur, ada beberapa hal yang bisa diantisipasi:

Pertama, ajarkan strategi dua model dari hari pertama: gunakan Gemini Flash untuk semua iterasi, eksperimen, dan debugging. Simpan Gemini Pro hanya untuk final build dan task yang membutuhkan reasoning paling dalam. Gemini Flash sekitar 8x lebih hemat dari Gemini Pro dalam penggunaan token.

Kedua, ajarkan murid konsep "checkpoint before major change" — setiap sebelum minta AI melakukan perubahan besar, buat checkpoint terlebih dahulu. Ini kebiasaan baik yang juga relevant di tools manapun.

Ketiga, siapkan fallback content jika ada downtime. Tools berbasis cloud bisa mengalami gangguan, dan jadikan ini pelajaran tentang pentingnya tidak bergantung pada satu tools saja.

---

## BAGIAN 7: REKOMENDASI PENGGUNAAN TOOLS DALAM KURIKULUM

Meskipun Antigravity sebagai tools utama, tools lain bisa diposisikan sebagai elemen kurikulum yang memberi murid perspektif yang lebih luas:

**Pertemuan 1–2 (Intro):** Kenalkan Lovable atau Emergent sebagai "contoh tools yang paling mudah" — biarkan murid merasakan sensasi membuat app dari prompt dalam 10 menit. Ini untuk membangun antusiasme dan membuktikan bahwa vibe coding nyata adanya. Sekaligus demonstrasikan limitasinya: kontrolnya rendah, token cepat habis.

**Pertemuan 3 sampai akhir (Core):** Pindah ke Antigravity sebagai tools utama. Ini di mana murid belajar cara kerja yang sesungguhnya: memahami codebase, bekerja dengan agent, iterasi yang terstruktur.

**Pertemuan terakhir (Outlook):** Perkenalkan Cursor dan Claude Code sebagai "tangga selanjutnya" — bukan untuk dipraktikkan, tapi murid tahu ada level berikutnya jika mereka ingin terus berkembang ke arah yang lebih teknis. Ini membuka horizon tanpa overwhelm.

## KESIMPULAN

Lanskap AI coding tools di 2026 sudah sangat mature dan terdiversifikasi. Untuk non-technical users yang ingin hasil instan, Lovable dan Emergent adalah paling mudah. Untuk developer profesional yang butuh kontrol penuh, Cursor dan Claude Code adalah pilihan terbaik.

Tapi untuk konteks mengajarkan non-technical people membangun produk digital yang nyata, mengembangkan kemampuan berpikir logis dan terstruktur, sekaligus mempersiapkan mereka untuk dunia digital yang sesungguhnya — Antigravity IDE berada di sweet spot yang tidak dimiliki tools lain.

Ditopang oleh ekosistem Google yang sudah terbukti dan terintegrasi, context window terbesar di kelasnya, pendekatan agent-first yang visual dan bisa diajarkan, harga yang kompetitif dengan value terbesar per rupiah, dan fondasi VS Code yang portabel — ini adalah pilihan yang defensible secara strategis, finansial, dan pedagogis.

Vibe coding bukan tentang menghilangkan kerja keras — tapi tentang memindahkan energi dari "menulis kode" ke "memikirkan produk." Dan itulah skill yang paling berharga untuk diajarkan kepada siapa pun yang ingin berkarya di era digital ini.
