## I. Komunikasi & Struktur Grup

### Prinsip Komunikasi

- **Segera responsif** — Minimal acknowledgment dalam 2 jam selama jam kerja 09:00-21:00
- **Tidak ada pertanyaan bodoh** — Budaya aman untuk bertanya digaungkan terus di setiap grup
- **Celebrate wins publik** — Bangun moral cohort dengan highlight achievement secara reguler
- **Personal untuk masalah individual** — Issues sensitif tidak dibawa ke grup
- **Recording & notes via grup** — Semua resource di-share lewat grup, bukan DM

### Struktur Grup — 3 Grup per Batch

Untuk setiap batch/program, buat **3 grup terpisah** dengan fungsi berbeda. Ini menghindari noise dan memastikan informasi penting tidak tenggelam.

---

**Aturan Grup (letakkan di description/pinned message):**

1. Pertanyaan teknis BOLEH kapan saja — akan dijawab max 24 jam (jam kerja)

3. Celebrate wins — kalau berhasil deploy, share di grup!

## V. Crash Course — Materi Detail

> Catatan: "Flash Course" diganti jadi "Crash Course" untuk resonance lebih kuat di pasar Indonesia.

### Identitas Crash Course

| Item      | Detail                                                       |
| --------- | ------------------------------------------------------------ |
| Nama      | Vibe Coding Crash Course                                     |
| Tagline   | "Bikin Website dari Nol dalam 90 Menit. Tanpa Coding."       |
| Durasi    | 90-120 menit                                                 |
| Target    | SMA / Mahasiswa / Profesional muda — tanpa pengalaman coding |
| Kapasitas | 50-200 peserta                                               |
| Harga     | Rp 50.000 atau GRATIS (top-of-funnel)                        |
| Outcome   | Setiap peserta punya 1 halaman website live di internet      |

### Persona Peserta

**Anisa, 17 tahun, SMA kelas 11**
Motivation: Lihat teman bikin portofolio online, ikut-ikutan tapi bingung mulai dari mana
Pain Point: Merasa tidak bisa karena "tidak jago teknologi"
Keberhasilan: Pulang dengan 1 halaman website live + tahu "ini untuk saya"

**Dimas, 19 tahun, mahasiswa semester 3 (non-IT)**
Motivation: Ingin skill marketable + dengar soal AI + coding
Pain Point: Bingung coding tradisional vs AI coding
Keberhasilan: Paham bedanya + punya website + siap daftar bootcamp

### Tujuan Pembelajaran

Di akhir crash course, peserta diharapkan:

1. **Memahami** konsep vibe coding — AI sebagai partner untuk membangun
2. **Menguasai** minimal 1 template prompt yang reusable
3. **Menerapkan** decomposition sederhana untuk memecah fitur jadi langkah kecil
4. **Memproduksi** 1 halaman website live di internet
5. **Mengevaluasi** output AI secara kritis

### Struktur Crash Course (120 Menit) — Detail per Menit

---

#### Bagian 1 — Hook & Reframing Mindset (15 menit)

**Menit 0-5: Demo Langsung**

- Mentor bangun landing page / halaman bio dari prompt ke live URL dalam <10 menit
- Ini HARUS nyata — tidak di-script sebelumnya
- Bahasa tubuh: confident, pace-nya cepat, explain sambil ketik

**Menit 5-10: Reframe**

- "Tadi yang kalian lihat — saya tidak mengetik satu baris kode pun"
- "Saya hanya menjelaskan apa yang saya mau ke AI, dan AI yang menulisnya"
- "Itulah yang namanya vibe coding"
- "Dan hari ini, kalian akan merasakan langsung"

**Menit 10-15: Urgensi**

- "Dalam 2-3 tahun ke depan, kemampuan ini akan seperti Microsoft Office — wajib"
- "Dan sekarang kamu punya kesempatan berada di depan 95% orang lain"
- "Asalkan kamu mau mencoba"

---

#### Bagian 2 — Core Teaching (30-35 menit)

**Menit 15-25: Problem Decomposition**

Demo: "Saya mau bikin aplikasi pencatat keuangan"

Tanpa decomposition: butuh 50 prompt yang tidak jelas, hasilnya berantakan

Dengan decomposition:

1. Apa yang mau disimpan? (nama transaksi, jumlah, tanggal, kategori)
2. Bagaimana data mengalir? (input → simpan → tampilkan → hapus/edit)
3. Apa yang perlu ditampilkan? (tabel, grafik ringkasan, filter)

- Latihan interaktif: minta peserta tulis "masalah sehari-hari + solusi digital" di chat
- Mentor pilih 1 dan dekomposisi secara live

**Menit 25-40: Prompt Specificity**

Before (vague):
"Buat tombol"

After (spesifik):
"Buat tombol biru bertuliskan 'Simpan' di pojok kanan bawah form. Saat diklik, muncul loading spinner 1 detik, lalu form kosong kembali dan data muncul di tabel di bawah. Jika koneksi error, tampilkan pesan merah: 'Gagal menyimpan. Coba lagi.'"

Empat level berpikir sebelum prompt:

1. Logical — apa yang sedang dibangun?
2. Analytical — apa tujuan utamanya?
3. Computational — bagaimana logikanya bekerja?
4. Procedural — strategi fitur apa yang dipakai?

Template prompt:
[KONTEKS] → [YANG DIINGINKAN] → [SPESIFIKASI] → [CONSTRAINT]

**Menit 40-50: Evaluasi Kritis Output AI**

"AI bisa salah. AI yang menjawab dengan percaya diri belum tentu benar."

Checklist evaluasi:

- Apakah output melakukan yang diminta?
- Apakah ada bagian yang tidak perlu dan bisa mengganggu?
- Apakah ada hardcode yang seharusnya dinamis?
- Jalankan dan test — jangan hanya lihat kode

---

#### Bagian 3 — Live Build Bersama (25-30 menit)

**Format:** Mentor tulis prompt di layar (share screen). Siswa ikut dengan project serupa.

**Menit 50-65: Build Halaman Bio**

- Mentor demonstrasikan: prompt → generate → iterate 1-2x
- Siswa ikut di laptop masing-masing
- Mentor berkeliling via "siapa yang sudah selesai?"

**Menit 65-75: Deploy Bareng**

- Mentor: GitHub repo → Vercel → deploy → live URL
- Siswa ikut step yang sama
- Momen celebration: "Semua orang di ruangan ini sekarang punya sesuatu yang live di internet"

**Menit 75-80: Modify & Iterate**

- Mentor: "sekarang coba ubah warnanya jadi hijau"
- Demonstrasi prompt iterasi: "sisanya jangan diubah"

---

#### Bagian 4 — Reality Check: Project vs Project Sesungguhnya (15 menit)

**Tujuan:** Tunjukkan kompleksitas nyata — bukan untuk menakut-nakuti, tapi untuk transparency.

Comparison:

```
Crash Course (hari ini):
  1 halaman statis
  Tanpa database
  Tanpa login
  Tidak menyimpan data

Project nyata yang useful:
  Multiple halaman + navigasi
  Database (menyimpan data permanen)
  Autentikasi (login personal)
  Fitur CRUD (tambah, ubah, hapus data)
  Mobile-friendly
  Testing + polish
```

Peta perjalanan yang jujur:
Planning (PRD, wireframe) → Database → Auth → Build → Test → Deploy → Maintenance

Narasi: "Hari ini kalian sudah merasakan fondasinya. Kalau mau bangun sesuatu yang lebih berguna — aplikasi pencatat keuangan yang menyimpan data, website komunitas dengan form pendaftaran — itu yang kita bangun di Builder Bootcamp."

---

#### Bagian 5 — Q&A + Soft Offer (15 menit)

**Menit 80-87: Q&A Dulu**

- "Ada pertanyaan sebelum kita bicara soal kelanjutan?"
- Biarkan peserta bertanya. Jawab dengan jujur.
- Jika ada pertanyaan teknis: jawab, tapi tutup dengan "di bootcamp kita bahas lebih dalam"

**Menit 87-95: Soft Offer**

- "Hari ini kalian sudah merasakan fondasinya. Kalau mau lanjut — Builder Bootcamp 8 sesi adalah pilihan utama."
- "Di situ kalian akan bangun aplikasi yang menyimpan data. Bukan hanya tampilan."
- Harga: Rp 350.000 untuk 4 sesi.
- Insentif: "Peserta crash course hari ini dapat early bird Rp 299.000 — berlaku sampai [H+3]."
- Tunjukkan hasil project dari alumni (screenshot, link jika ada)

**Menit 95-100: Call to Action**

- "Link registrasi ada di pinned message."
- "Kalau belum yakin — tidak apa-apa. Yang penting kalian pulang dengan sesuatu hari ini."
- "Dan kalau nanti berubah pikiran, batch berikutnya akan di-announce di grup ini."

**Menit 100-120: Open Chat / Networking**

- Biarkan peserta chat, tanya satu sama lain, mentor available
- Ini optional tapi membangun sense of community

## X-A. Communication Backup & Contingency

### Backup Communication Channel

**Mengapa perlu:** Telegram atau WhatsApp bisa down, grup di-banned, atau nomor admin ter-blok. Peserta yang kehilangan akses ke grup utama akan hilang dari radar.

**Primary:** Telegram Group (VC | [Batch] | Main)
**Backup 1:** WhatsApp Group (dibuat SAAT registrasi — WhatsApp sebagai duplikat pasif, di-mute sampai diaktifkan)
**Backup 2:** Email broadcast (Mailchimp) — untuk announcement kritis
**Backup 3:** Google Sites landing page dengan changelog/announcement

**Aturan implementasi backup:**

1. **Saat registrasi:** minta email AND nomor WhatsApp. WhatsApp group adalah backup pasif — peserta di-add tapi grup dalam mode mute/shuffle sampai diaktifkan
2. **Telegram down/unavailable:** switch ke WhatsApp group. Kirim pesan: "Telegram sedang gangguan. Update terbaru di WhatsApp Group ini mulai sekarang."
3. **WhatsApp di-banned/nomor blocked:** email broadcast + Google Sites page
4. **Semua platform down:** nomor WA admin sebagai contact point personal

**Critical Communication Template (untuk semua platform):**

```
[URGEN] Update Penting — Vibe Coding [Batch]

[Isi announcement]

Jika kamu tidak bisa mengakses grup utama:
- WhatsApp: [link grup]
- Email: [alamat email]
- Kontak langsung: [nomor WA]

— Tim Vibe Coding
```

---

### Zoom Failure Contingency Plan

**Risiko:** Zoom down saat sesi akan dimulai, Zoom meeting di-kick, koneksi mentor tidak stabil.

**Backup Plan — Tiered Response:**

**Tier 1 (Zoom error saat sudah berjalan):**

- Kirim pesan di grup: "Zoom sedang error, switch ke Google Meet"
- Share Google Meet link dari pinned message
- Minta TA di grup untuk bantu pantau chat
- Estimasi downtime: 2–5 menit

**Tier 2 (Zoom not accessible dari awal):**

- Langsung switch ke Google Meet
- Google Meet link selalu ada di pinned message sebagai backup
- Jika Google Meet juga error: gunakan Jitsi (jitsi.org — no account needed)
- Jika mentor yang connectivity issue: TA handle sementara, cari pengganti atau reschedule

**Google Meet sebagai Backup Permanent:**

- Buat Google Calendar event dengan Google Meet link untuk setiap sesi
- Meet link = backup link, zoom link = primary link
- Kedua link ada di pinned message

**Recording Backup:**

- Zoom cloud recording = primary
- Zoom local recording = backup (laptop mentor)
- Jika Zoom cloud down: upload local recording manual ke Google Drive dalam 24 jam

**Reschedule Threshold:**

- Jika >30% peserta tidak bisa join dalam 15 menit pertama karena platform error → reschedule sesi
- Jika mentor connectivity yang bermasalah → offered waktu alternatif dalam 48 jam

## X-D. Refund & Cancellation Policy

### Kebijakan Umum

Semua policy ini harus dicantumkan di halaman registrasi SEBELUM pembayaran dilakukan. Peserta yang sudah bayar = dianggap sudah membaca dan menyetujui policy ini.

| ----------------------------------------------------- | ---------------------------- |
| Peserta cancel sebelum H-1                            | Refund penuh                 |
| Peserta cancel <H-1                                   | Tidak ada refund             |

| Batch dibatalkan (tidak mencapai quorum min. 5 orang) | Refund penuh                 |

---

### Format Online — Bootcamp (4 / 8 / 12 Sesi)

| Situasi                                           | Policy                                                           |
| ------------------------------------------------- | ---------------------------------------------------------------- |
| Peserta cancel sebelum Sesi 1 dimulai             | Refund penuh (dikurangi biaya processing 3%)                     |
| Peserta cancel setelah Sesi 1 tapi sebelum Sesi 2 | Refund 75% dari total biaya                                      |
| Peserta cancel setelah Sesi 2 dimulai             | Tidak ada refund                                                 |
| Peserta drop out mid-program                      | Tidak ada refund untuk sesi yang sudah berjalan                  |
| Mentor cancel (alasan personal)                   | Replacement dengan mentor cadangan, atau refund proporsional     |
| Batch dibatalkan (tidak mencapai quorum)          | Refund penuh 100%                                                |
| Batch postpone (>7 hari)                          | Pilihan: refund penuh atau automatic enrollment batch berikutnya |

**Quorum minimum:** 5 peserta per batch. Jika kurang dari 5 pendar 48 jam sebelum Sesi 1, batch digabung dengan batch berikutnya atau di-refund.

---

### Format Online — Recording Policy

- Recording disediakan untuk peserta terdaftar saja
- Recording dikirim max 48 jam setelah sesi selesai
- Recording berlaku selama program + 30 hari setelah selesai
- Recording tidak tersedia untuk pembelian individual — hanya untuk peserta yang sudah terdaftar
- Mentor berhak exclude bagian tertentu jika ada alasan (diskusi personal, teknis)

---

### Format Private / Small Group

| Situasi                                     | Policy                                                                     |
| ------------------------------------------- | -------------------------------------------------------------------------- |
| Reschedule                                  | Maks 1x per program, min. 24 jam sebelumnya. <24 jam = sesi tetap dihitung |
| Mentor tidak bisa hadir                     | Replacement atau refund proporsional untuk sesi yang tidak berjalan        |
| Peserta cancel sebelum Sesi 1               | Refund penuh (dikurangi 3% processing fee)                                 |
| Peserta cancel mid-program (sebelum 4 sesi) | Tidak ada refund untuk sesi yang sudah berjalan                            |
| Peserta cancel mid-program (setelah 4 sesi) | Sisa sesi tidak wajib dibayar                                              |
| Force majeure (bencana, pandemi)            | Refund proporsional atau freeze program                                    |

---

### Format Sekolah

| Situasi                                           | Policy                                                                                  |
| ------------------------------------------------- | --------------------------------------------------------------------------------------- |
| Sekolah cancel sebelum program dimulai            | Pembayaran untuk sesi yang sudah berjalan saja                                          |
| Sekolah cancel mid-program (setelah sesi dimulai) | Pembayaran untuk semua sesi yang sudah berjalan + 1 sesi penalti                        |
| Mentor tidak bisa hadir (sebab personal)          | Replacement atau sesi pengganti dalam 7 hari                                            |
| Siswa individual drop out                         | Urusan antara sekolah dan siswa — tidak ada refund ke siswa individual dari sisi mentor |
| Minimum quorum siswa per kelas                    | 10 siswa. Di bawah 10, program tetap berjalan dengan harga yang dinegosiasikan ulang    |

---

### Cara Request Refund

1. Email ke [alamat email] dengan subjek: "Refund Request — [Nama] — [Batch/Program]"
2. Isi: nama, program, tanggal registrasi, alasan refund
3. TA respond dalam 24 jam (jam kerja)
4. Refund diproses dalam 5–10 hari kerja ke rekening/e-wallet peserta
