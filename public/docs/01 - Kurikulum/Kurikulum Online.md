# Kurikulum Vibe Coding | Overcode — Format Online

> **Brand:** Overcode

### Panduan untuk Mentor: Crash Course & Builder Bootcamp

**Target:** Audiens umum (SMA–mahasiswa–profesional muda), cold audience
**Format:** Cohort-based, live online sessions via Zoom/Google Meet
**Pricing:**

- Crash Course (1 sesi × 2–2.5 jam): Rp 50.000
- Builder Bootcamp (8 sesi × 2–2.5 jam): Rp 600.000

---

## Overview

Format online memiliki dinamika berbeda dari Private dan Sekolah. Audiens adalah orang asing yang belum kenal mentor, tidak ada captive audience, dan perhatian harus terus dijaga. Kesuksesan bergantung pada tiga hal:

1. **Flash Course yang jujur dan powerful** — membangun trust dan taste of value
2. **Onboarding teknis yang solid** — siswa harus siap sebelum Sesi 1
3. **Cohort community** — siswa membantu satu sama lain, bukan hanya bergantung pada mentor

Kurikulum ini mencakup **Crash Course** (1 sesi) dan **Builder Bootcamp** (8 sesi). Detail Builder Bootcamp penuh ada di file terpisah: `Vibe Coding - Kurikulum Builder Bootcamp.md`.

## I. Filosofi & Prinsip Dasar

Sama dengan Private dan Sekolah — perbedaannya ada di **pengalaman delivery**, bukan substance.

| Prinsip                     | Penjelasan untuk Konteks Online                                                                                                                |
| --------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| **Problem-First**           | Project harus dipilih sendiri oleh siswa — ini lebih penting di online karena tidak ada mentor yang bisa memastikan relevance secara personal. |
| **Self-Contained Value**    | Flash Course harus mengajarkan sesuatu yang berguna meski siswa tidak lanjut ke bootcamp.                                                      |
| **Open Loop yang Jujur**    | Tunjukkan kompleksitas nyata di akhir Flash Course — bukan untuk FOMO, tapi untuk transparency.                                                |
| **Community as Multiplier** | Di format online tanpa 1-on-1, peer support adalah force multiplier. Bangun ini dari hari pertama.                                             |

### Perbedaan Utama dengan Format Private dan Sekolah

| Aspek              | Private          | Sekolah        | Online                              |
| ------------------ | ---------------- | -------------- | ----------------------------------- |
| Rasio mentor:siswa | 1:1 atau 1:3–5   | ~1:15–20       | ~1:10–15 (cohort)                   |
| Kehadiran          | Wajib            | Wajib (ekskul) | Voluntary — perlu insentif          |
| Setup teknis       | Mentor bantu     | Lab komputer   | Siswa sendiri, harus siap dari awal |
| Debugging          | Real-time 1-on-1 | Klasikal       | Async (grup chat) + sync (sesi)     |
| Community          | Personal         | Kelas          | Grup chat — perlu difasilitasi      |
| Recording          | Tidak perlu      | Opsional       | Wajib — nilai tambah utama          |

## II. Technical Pre-Flight Checklist

Ini adalah langkah paling penting di format online. Siswa harus setup semua tools sendiri di rumah — tanpa ini, Sesi 1 akan habis untuk troubleshooting teknis.

### Timeline Onboarding

**H+0 (saat registrasi):**

- Kirim email terima kasih + link invoice
- Lampirkan **Technical Setup Guide** (video 5 menit: cara daftar Antigravity, GitHub, Vercel)
- Link ke Google Form: "Pre-flight Checklist"
- Minta siswa join grup WhatsApp/Discord cohort

**H+3 (3 hari sebelum Sesi 1):**

- Reminder: "Sesi 1 dimulai [hari/tanggal]. Pastikan semua tools sudah terinstall."
- Google Form reminder: "Sudah berhasil daftar Antigravity? Ya / Belum / Stuck di mana"
- Jika ada yang stuck: DM personal untuk bantu resolve sebelum sesi

**H+1 (1 hari sebelum Sesi 1):**

- Kirim link Zoom/Meet
- Kirim reminder: recording akan dikirim max 48 jam setelah sesi
- Minta siswa test koneksi, audio, dan kamera sebelum sesi

### Technical Setup Guide (Harus Dikirim Saat Registrasi)

**Langkah 1 — Akun yang Perlu Dibuat:**

1. **GitHub** → github.com → Sign up → free plan
2. **GitHub Desktop** → desktop.github.com → download & install
3. **Antigravity** → daftar di antigravity.id (atau link yang sesuai)
4. **Vercel** → vercel.com → Sign up with GitHub
5. **Gemini Advanced** → aktifkanlangganan (atau alternatif AI yang disepakati)

**Langkah 2 — Verifikasi:**

- Buka Antigravity — apakah bisa diakses?
- Buka GitHub — apakah sudah login?
- Buka Vercel — apakah sudah terkoneksi dengan GitHub?
- Buka Gemini Advanced — apakah berfungsi?

**Langkah 3 — Masalah Umum:**

| Masalah                           | Solusi                                                            |
| --------------------------------- | ----------------------------------------------------------------- |
| Antigravity tidak bisa diakses    | Coba VPN atau browser lain (Chrome/Edge)                          |
| GitHub Desktop error saat install | Unduh ulang dari desktop.github.com                               |
| Vercel tidak connect ke GitHub    | Pastikan GitHub sudah login di browser yang sama                  |
| Gemini Advanced tidak aktif       | Hubungi support atau gunakan ChatGPT sebagai alternatif sementara |

> Jika siswa tidak bisa menyelesaikan setup sebelum Sesi 1, mereka akan kehilangan 15–20 menit pertama sesi. Dorong untuk menyelesaikan ini sejak hari pertama registrasi.

## III. Crash Course — 1 Sesi

**Target:** 50–100+ peserta per batch
**Durasi:** 2–2.5 jam
**Harga:** Rp 50.000 atau gratis sebagai top-of-funnel
**Tujuan:** Edukasi + taste of value + soft-sell Builder Bootcamp

> Flash Course adalah sesi paling penting dalam seluruh funnel. Desain yang tepat menghasilkan konversi tinggi ke Starter Bootcamp. Desain yang salah menghasilkan kepuasan semu yang tidak menghasilkan aksi.

### Prinsip Desain Flash Course

**Harus memenuhi DUA kebutuhan sekaligus:**

1. **Self-contained value** — peserta yang hanya ikut 1 sesi ini dan tidak lanjut, tetap harus pulang dengan insight nyata yang bisa langsung dipakai.
2. **Open loop yang jujur** — di akhir sesi, peserta secara natural sadar bahwa "membangun 1 hal kecil" sangat berbeda dari "membangun aplikasi yang utuh." Ini transparansi, bukan manipulasi.

### Struktur Flash Course (120 Menit)

---

#### Bagian 1 — Hook & Reframing Mindset (15 menit)

**Tujuan:** Capture attention, ubah definisi "ngoding" di kepala peserta.

**Aksi mentor:**

1. (Menit 0–5) **Demo langsung:** Bangun landing page / halaman bio dari prompt ke live URL dalam <10 menit. Tidak ada yang di-script sebelumnya. Ini harus nyata.

2. (Menit 5–10) **Reframe:** "Tadi yang kalian lihat — saya tidak mengetik satu baris kode pun. Saya hanya menjelaskan apa yang saya mau ke AI, dan AI yang menulisnya. Itulah yang namanya vibe coding."

3. (Menit 10–15) **Urgensi:** "Dalam 2–3 tahun ke depan, kemampuan ini akan menjadi seperti kemampuan mengoperasikan Microsoft Office — wajib, bukan opsional. Dan sekarang, kamu punya kesempatan untuk berada di depan 95% orang lain."

---

#### Bagian 2 — Core Teaching: Computational Thinking & Prompting (30–35 menit)

**Tujuan:** Berikan kerangka berpikir yang bisa dipakai peserta hari itu juga, tanpa perlu device.

**Topik 1: Problem Decomposition (10 menit)**

- Demo: "Saya ingin buat aplikasi pencatat keuangan" — tanpa decomposition ini akan butuh 50 prompt yang tidak jelas
- Latihan interaktif (di chat): minta peserta tulis "masalah sehari-hari + solusi digital yang mereka bayangkan" di chat — lalu mentor dekomposisi salah satu secara live

**Topik 2: Prompt Specificity (15 menit)**

- Before/after: "Buat tombol" vs. "Buat tombol biru bertuliskan 'Simpan' di pojok kanan bawah form, dengan loading state saat diklik"
- Empat level berpikir: Logical → Analytical → Computational → Procedural
- Template prompt: [KONTEKS] → [YANG DIINGINKAN] → [SPESIFIKASI] → [CONSTRAINT]

**Topik 3: Evaluasi Kritis Output AI (10 menit)**

- "AI bisa salah. AI yang menjawab dengan percaya diri belum tentu benar."
- Checklist: Apakah ini melakukan yang diminta? Apakah ada bagian yang tidak perlu? Jalankan dan test — jangan hanya lihat kode.

---

#### Bagian 3 — Live Build Bersama (25–30 menit)

**Tujuan:** Beri peserta pengalaman taktil "aku baru saja membuat sesuatu yang hidup di internet."

**Format:** Peserta ikut serta di device masing-masing, tapi mentor yang menulis prompt di layar dan menjelaskan setiap langkah. Peserta follow di laptop/HP mereka.

**Project:** Halaman bio / landing page 1 halaman — sangat sederhana, selesai dalam 25 menit.

**Langkah:**

1. Mentor demonstrasikan: tulis prompt, generate, iterate
2. Peserta ikut dengan project serupa (dengan topik masing-masing)
3. Deploy ke Vercel bersama-sama
4. Momen celebration: "Semua orang di ruangan ini sekarang punya sesuatu yang live di internet."

> Jika bandwidth memungkinkan, gunakan **breakout rooms** (2–3 orang) untuk diskusi. Tapi jangan sampai peserta merasa ditinggalkan tanpa panduan.

---

#### Bagian 4 — Reality Check: Apa yang Beda di Project Sesungguhnya (15 menit)

**Tujuan:** Tunjukkan kompleksitas nyata dengan jujur — bukan untuk menakut-nakuti, tapi untuk membangun ekspektasi yang realistis.

**Visual comparison (di layar):**

```
Flash Course (hari ini):
  → 1 halaman statis
  → Tanpa database
  → Tanpa login
  → Tidak menyimpan data

Project nyata yang useful:
  → Multiple halaman + navigasi
  → Database (menyimpan data permanen)
  → Autentikasi (login personal)
  → Fitur CRUD (tambah, ubah, hapus data)
  → Mobile-friendly
  → Testing + polish
```

Jelaskan peta perjalanan: Planning (PRD, wireframe) → Database → Auth → Build → Test → Deploy → Maintenance.

---

#### Bagian 5 — Q&A + Soft Offer (15 menit)

**Tahap 1 — Q&A dulu (7–8 menit):**

- "Ada pertanyaan sebelum kita bicara soal kelanjutan?"
- Biarkan peserta bertanya. Ini membangun trust.
- Jawab dengan jujur — termasuk tentang keterbatasan dan apa yang perlu diperbaiki.

**Tahap 2 — Soft offer (7–8 menit):**

- "Hari ini kalian sudah merasakan fondasinya. Kalau kalian mau bangun sesuatu yang lebih useful — misalnya aplikasi pencatat keuangan yang menyimpan data, atau website komunitas dengan form pendaftaran — itu yang kita bangun di Builder Bootcamp."
- Tunjukkan hasil project dari alumni (screenshot, link jika ada).
- Harga: Rp 600.000 untuk 8 sesi.
- Insentif terbatas waktu: "Peserta flash course hari ini mendapat harga early bird Rp 299.000 — berlaku sampai [H+3]."
- Jangan paksa. Cukup出示 options dan biarkan peserta decide.

---

### Checklist Flash Course Berhasil

- [ ] Minimal 1 kerangka berpikir yang transferable (prompt engineering / decomposition)
- [ ] Semua peserta punya 1 halaman live di internet
- [ ] Peserta bisa jelaskan perbedaan "website statis" vs "web app dengan database"
- [ ] Peserta tahu apakah mereka mau lanjut ke bootcamp atau tidak (ini penting — kejujuran)
- [ ] Tidak ada yang merasa "dijebak" untuk bayar

## V. Cohort Management

### Grup Komunikasi

Buat **WhatsApp group atau Discord** untuk setiap cohort. Aturan grup:

- Pertanyaan teknis boleh kapan saja — mentor/TA akan respond dalam 24 jam (selama jam kerja)
- Tidak ada pertanyaan "bodoh" — semua orang pernah mulai dari nol
- Tidak ada spam, tidak ada off-topic
- Celebrate wins: kalau ada yang berhasil deploy, share di grup

### Progress Tracker

Gunakan Google Sheets yang di-share ke seluruh cohort:

| Nama   | Project | S1  | S2  | S3  | S4  | Milestone   | Notes |
| ------ | ------- | --- | --- | --- | --- | ----------- | ----- |
| [Nama] | [Judul] | ✓/✗ | ✓/✗ | ✓/✗ | ✓/✗ | M1/M2/M3/M4 |       |

Update tracker setiap akhir sesi.

### Recording Policy

- Recording tersedia untuk peserta yang terdaftar (bukan untuk yang tidak daftar)
- Recording dikirim max 48 jam setelah sesi via link Google Drive
- Recording berlaku selama program + 30 hari setelah selesai
- Mentor bisa exclude bagian tertentu jika ada alasan

## VI. Dropout & Refund Policy

**Komitmen:** 4 sesi dibayar di awal (Starter), 8 sesi (Builder), 12 sesi (Pro)

**Dropout mid-program:**

- Tidak ada refund untuk sesi yang sudah berjalan
- Recording sesi sebelumnya tetap tersedia
- Peserta bisa join cohort berikutnya (tanpa biaya tambahan) jika ingin catch up

**Batch cancellation:**

- Jika < 5 peserta registrasi: batch dibatalkan, refund penuh atau offered ke batch berikutnya
- Jika mentor yang batal: offered waktu alternatif atau refund proporsional

**Reschedule:**

- Jika mentor perlu reschedule: informed min. 48 jam sebelumnya, offered waktu alternatif
- Jika peserta tidak bisa hadir: gunakan recording, tidak ada pengganti sesi individual

## VIII. Instructor Load & Capacity

Untuk 1 mentor full-time, kapasitas realistis per bulan:

| Aktivitas                             | Waktu per Minggu                                | Catatan             |
| ------------------------------------- | ----------------------------------------------- | ------------------- |
| Crash Course (50–100+ peserta)       | ~3 jam (prep + pelaksanaan)                     | Batch besar, 1 sesi |
| Builder Bootcamp (10–15 siswa/cohort) | ~8 jam/minggu                                   | 8 sesi = 8 minggu   |
| Private mentoring (3 siswa)           | ~6 jam/minggu                                   |                     |
| Ekskul Sekolah (1 kelas)              | ~4 jam/minggu                                   |                     |

**Maks realistic load:** 1 Crash Course + 1 Builder Bootcamp + 1 ekskul sekolah secara simultan.
