# Script Materi: Pertemuan 3 - 02. Git Commit sebagai "Save Game" (Subbab 1.1)

## 📖 Bab 1: Mengapa Kita Butuh Mesin Waktu?

### Subbab 1.1: Analogi "Save Game" di Konsol Game vs Git Commit

Di dunia pemrograman, ada satu alat luar biasa bernama **Git**. Git adalah sistem pelacak perubahan berkas yang bekerja di balik layar komputer kita. Untuk memahaminya tanpa pusing, mari kita gunakan **Analogi Bermain Game**:

Bayangkan Anda sedang memainkan game petualangan yang sangat seru dan menantang (seperti GTA, Resident Evil, atau game RPG lainnya). 

Di dalam game tersebut, sebelum Anda masuk ke area pertempuran melawan monster bos yang sangat besar dan berbahaya, apa yang selalu refleks Anda lakukan pertama kali? 

Benar! Anda akan mencari **titik penyelamatan** (*Save Point*) dan melakukan **Save Game** (menyimpan game). 

Kenapa Anda melakukan itu? 
Karena Anda tahu, jika nanti Anda masuk ke sarang monster lalu karakter Anda kalah dan gugur, Anda tidak perlu mengulang permainan dari level 1 yang memakan waktu berhari-hari. Anda tinggal memuat ulang (*Load Game*) dari titik penyimpanan terakhir Anda yang aman, tepat sebelum Anda melawan monster tersebut. Anda bisa mencoba taktik baru dengan tenang.

---

### Konsep Git Commit: Foto Kenangan Proyek

Di dunia pembuatan aplikasi, melakukan penyimpanan aman ini disebut dengan **Git Commit**.

Setiap kali Anda berhasil membuat satu fitur website berjalan dengan lancar (misalnya: Anda sukses membuat Navbar atas terlihat rapi), Anda melakukan *Commit*. Anda menyuruh Git untuk mengambil "foto kenangan" atau titik aman dari seluruh berkas proyek Anda saat itu juga. 

Anda memberi label pada foto tersebut, misalnya: *"Commit 1: Navbar Selesai dan Rapi"*.

Kemudian, Anda lanjut bekerja membuat bagian kartu profil. Tiba-tiba, asisten AI salah merakit kode dan membuat seluruh website Anda macet total, tombol-tombol hilang, dan layar berubah menjadi putih kosong. 

Bagi orang yang tidak menggunakan Git, ini adalah kiamat kecil. Mereka akan stres, mencoba menghapus kode baris demi baris, dan sering kali berakhir dengan membuat kodenya semakin hancur berantakan.

Namun, bagi seorang Vibe Coder yang disiplin melakukan *Commit*, ini hanyalah masalah sepele. Anda cukup menyuruh Git: *"Tolong muat kembali (Load) titik aman 'Commit 1' kemarin."*

Dalam satu detik, seluruh berkas proyek Anda akan kembali persis seperti saat Navbar selesai dibuat. Kode yang rusak akibat kesalahan AI terhapus bersih, dan Anda bisa mulai memberikan perintah baru dengan tenang. 

Ingat prinsipnya: **Git Commit adalah fitur 'Save Game' proyek Anda yang menyelamatkan Anda dari frustrasi akibat error!**
